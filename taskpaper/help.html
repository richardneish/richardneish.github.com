Taskpaper.web

A simple, easy to use plain text file-based task management web application. It offers superb ways of filtering, grouping, searching for and managing your tasks. But ultimately 'all your tasks belong you', so everything is stored in a simple text file format. Tasks are created and edited directly in the text file, either in the application or in an editor of your choice.

Add tasks, projects and notes to quickly create a simple, easy-on-the-eye list of thing to do!

Easily add tags to any task (or even project), to help you find and group your list

Divide your tasks up into simple projects. Click to view that project only

Powerful search box, with Google-like syntax. Just type words or dates to look for, or use any of the special commands. For example, type '=late' to see tasks that are overdue.

Click the Edit button to edit your task list...it's just a plain text file underneath its shiny skin!

The Taskpaper Syntax

Projects

Projects are created by writing the project name on a line followed by a colon ( : ) 

This is a project:

Tasks

You make a task by beginning the line with dash-space ( -  ) 

- this is a task

Tags

No task management is complete without ubiquitious 'tags', a quick, simple way of group and filtering your tasks.  A tag is added by putting a @ symbol before the tag word 

- this is a task @tag @another-tag

(Note: sorry, no spaces allowed in tags...you'll get used to it!)

Dated Tags

You an also make a dated tag (i.e. a due date) by using the @ followed by the date 

- a dated tasks @15-dec-2009

(Note: the date format default is dd-mmm-yyyy, but this can be changed in the config.php file)

Notes

Any line beginning with ... (an ellipsis) is considered as a note. Simple as that.  Notes belong to the task above them 

...a note all by itself

Complete/Done

To show that a task is done, just put an X at the start of the line.  Simple, visible, memorable! 

X- this task is done

Highlighting/Starred

To highlight a task (aka: starred) you can simply put an * (asterisk) at the end of the line 

- this task is highlighted*

Other points worth noting:

It's usually best to leave a blank line between projects, it makes it easier to recognise the different projects.  At the top of the list is a good place to put any new (orphaned) tasks that you haven't had time to file yet.  As noted below tasks entered in the 'Search Box' also puts new tasks here by default.  

Adding and Editing Tasks

You can edit the whole taskpaper document by clicking the 'Edit' button on the toolbar at the top; make any edits you want and click 'Save' button when you're finished.  

You can also quickly add new tasks using the 'Search/Command Box' at the top right.  Just type a task as you normally would (i.e. start with '- ') and press enter when you're done.  You can also add tags, or even choose the project by typing '/' followed by the project number (Refer to the project numbers in list on left) or by selecting a project before you enter a new task.  By default all new tasks added this way will be put at the top if no project is given.

You can also use time periods in tags when adding tasks through the Command Box; for example if you type "@today" it will be changed to today's date when the task is added. The following time periods are recognised: 

@today, @tomorrow, @day, @week, @month, @year

Before the last 4 periods you can also put a number, e.g: 

@2day, @3week (i.e. in 2 days, in 3 weeks)

Note: spaces and plurals are not recognised (yet...)

Examples

So, to put it all together here's a complete example: 

The first project:
- get to work early @remind @important *
- get to bed early @maybe
X- buy flowers for your wife @remind @30-dec-2009

Second project:
- wake up early @soon
- eat wholemeal porridge @food @today*
...try Tiger Oats?

Group, Filter, Search

Click on any tag, anywhere to filter/group by that tag.  Same thing for projects; click on the project heading to see tasks from that project only.

Use the search box on the top right to do a full-text search, multiple words can be used.   Words can be excluded by putting '-' before them: 

yes -no

You can also search by date: 

>12-feb-2009 => all tasks after this date
<12-jan-2008 => all tasks before this date
"12-jan-2009 12-feb-2009" => all tasks between these dates
12-feb-2009..30-may-2009 => (alternative way of doing date ranges)

Or by time period: any of the following are accepted and understood: 

 =today, =week, =month, =quarter, =year

Use the special 'done', 'starred', 'due', 'late' tags to filter by these categories.

Of course you can also use these tags in the Search Box too: 

=due, =late, =star, =done

You can search for specific phrases by enclosing them in quotes (" "): 

"specific phrase"

All the above can be combined in one search as well, e.g.: 

=due order "door hinges"

Taskpaper will try to sort and group date searches by month for a better overview.   If your search contains many words, commands and/or dates, then put the date at the end to group the search. For example these searches would be grouped: 

roof =month
joe door =due

These searches will not be grouped: 

=month order
=due electrical
